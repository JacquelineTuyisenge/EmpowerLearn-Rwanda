HTML




HTML stands for HyperText Markup Language. It is the standard markup language used for creating and structuring web pages and web applications. HTML uses a system of tags and elements to annotate text, images, multimedia, and other content to describe the structure of a web page.

HTML provides a framework to define the different parts of a web page, such as headings, paragraphs, links, images, tables, forms, and more. These elements are structured within the HTML document using tags, which are enclosed within angle brackets <>, and at the end for closing tag is like <.../>. but some tags does not have closing tag because tgey close themselves: for example like: meta, which goes in head tag, link tag which goes in head tag too, and others.

Web browsers interpret HTML documents and render them into visible web pages that users can view and interact with on the internet. HTML works alongside other technologies like Cascading Style Sheets (CSS) for styling and layout, and JavaScript for adding interactivity to create dynamic web experiences.

HTML tags and what they do

<html> : This tag is used to define the root element of an HTML document.

<head> : This tag is used to define the head section of an HTML document, it contains meat information about the document such as title, links, and other metadata.

<title> : this contains the tile of a web page.

<body> : This tag is used to define the body section of an HTML document, it contains the content of the web page.

<h1> : This tag is used to define the heading of a web page. they start from h1 up to h6 and that is how the size decreases. means that h1 is bigger than h6.

<p> : This tag is used to define the paragraph of a web page, they can be many paragraph tags in heading tag.

<img> : This tag is used to define the image of a web page.

<a> : This tag is used to define the link of a web page.

<ul> : This tag is used to define the unordered list of a web page.

<ol> : This tag is used to define the ordered list of a web page.

<li> : This tag is used to define the list item of a web page.

<table> : This tag is used to define the table of a web page.

<tr> : This tag is used to define the table row of a web page.

<td> : This tag is used to define the table data of a web page.

<th> : This tag is used to define the table header of a web page.

<form> : This tag is used to define the form of a web page.

<input> : This tag is used to define the input of a web page.

<span> : This tag is used to define the span of a web page.

CSS stands for Cascading Style Sheets. It is a styling language used to describe the presentation and layout of HTML documents. CSS allows web developers to control the visual appearance of web pages, including the design, colors, layout, fonts, and more. Here are some key points about CSS:

Basic structure of css
  1. Selectors: CSS selectors target HTML elements based on their type, class, ID, attributes, or relationship with other elements.
  2. Properties: CSS properties control the appearance and behavior of the selected elements(e.g: colors, dont-size, margin,..)
  3. Values: CSS values are the actual values that are assigned to the properties of the selected elements ()e.g: hex colors, pixels values, percentages, vh and others .

How is it Used?

CSS is used to style the HTML elements of a web page.

CSS can be applied to any HTML element, they can be applied directly within an html document using <style> tag inside <head> section, and they can be stored in separate ".css" files and be linked to html documents using the <link> tag. this allows multiple pages to sahre the same styles

Git & GitHub

Git is a distributed version control system used for tracking changes in source code during software development. It allows multiple developers to collaborate on projects, enabling them to work on different versions of files, merge changes, and maintain a complete history of revisions.


GitHub, on the other hand, is a web-based hosting service and platform that utilizes Git for version control. It provides a graphical interface and additional features like issue tracking, pull requests, code review tools, and project management capabilities on top of Git. GitHub serves as a central hub for collaboration, allowing developers to host Git repositories, share code, contribute to open-source projects, and collaborate seamlessly with others, enhancing team productivity and facilitating open-source community contributions.

Basic Git commands

Initialize a Repository:

  • 'git init': Initializes a new Git repository in the current directory.

Check Status

  • 'git status': Checks the status of the current Git repository.

Stage Changes

  • 'git add .': Adds all changes to the staging area.
    OR
  • 'git add [file]': Adds the specified file to the staging area.

Commit Changes

  • 'git commit -m [message]': Commits the staged changes with a message.

View Commits

  • 'git log': Views the commit history.

create Branches

  • 'git branch [name]': Creates a new branch with the specified name.
  • 'git checkout [name]': Switches to the specified branch.

Merge Branches

  • 'git merge [name]': Merges the specified branch into the current branch.

Remote Repositories

  • 'git remote add [name] [url]': Adds a remote repository with the specified name and URL.
  • git push -u origin [branch_name]: Pushes local commits to a remote repository

Updating Local Repository:

  • 'git pull [remote] [branch]': Pulls changes from a remote repository.




Getting Started with GitHub

Create an account on GitHub


Setting up Git Configuration with your username and user email

git config --global user.name "Your Name"

git config --global user.email "Your Email"

Creating a new repository

  • Click on the '+' icon in the top right corner and select "New repository
  • Give it a name, add a description, choose public/private, and click "Create repository.

. Initializing a Local Repository and Linking to GitHub:
From Command Line:

  • Navigate to the local directory you want to initialize with Git
  • git init
  • Link your local repository to the remote one created on GitHub: :git remote add origin "github_repository_url"
  • Stage and commit changes as needed, then ush changes to Github :git push -u origin master

Collaborating on GitHub:

Cloning a Repository

  • git clone "github_repository_url: this Dowloads a repository to your local machine."

Forking a Repository

forking a repository is the process of creating a copy of an existing repository on GitHub.: Click the "Fork" button on a repository to create your copy.

Creating a Pull Request

Creating a Pull Request is the process of merging changes from your local repository into the remote repository.
After making changes in your forked repository, create a pull request to propose changes to the original repository.
